# Спецификация: редизайн веб-дашборда daily-digest

> Задача для Claude Code: переделать веб-дашборд (`app/render.py`) в новый дизайн
> «тёплый светлый минимализм» (bento-сетка, мягкие карточки, drag-to-reorder),
> добавить карточку крипты с переключателем валюты ₽/$ и новый источник данных.
> Дизайн уже согласован с владельцем — этот документ фиксирует результат.

## 0. Эталон и контекст

- **Визуальный эталон:** [`docs/dashboard-mockup.html`](dashboard-mockup.html) — рабочий
  статический мокап на реальной вёрстке. Открой его в браузере: это и есть целевой вид.
  Вся вёрстка/CSS/JS ниже — оттуда; переноси максимально близко.
- **Что переделываем:** функцию `render_html(d)` в `app/render.py` (сейчас тёмные карточки,
  градиентный greeting — устарело). Текст для Telegram (`render_telegram`) НЕ трогаем.
- **Соблюдай конвенции проекта** (`CLAUDE.md` в корне): server-rendered HTML, **без тяжёлых
  SDK и без build-пайплайна** (чистый CSS + vanilla JS), внешние вызовы через `requests`,
  **graceful degradation** (любой источник может упасть — карточка показывает пустое/ошибку,
  страница не падает), все настройки через `.env`/`config`, язык — русский, обращение на «ты».
- Ключевые продуктовые решения уже приняты владельцем — см. §9 «Принятые решения».
  Следуй интерактивному ревью из корневого `CLAUDE.md`, только если по ходу всплывут
  новые развилки, не покрытые этим документом.

## 1. Дизайн-система (CSS-токены — взять из мокапа)

```css
--bg:#efece5;        /* тёплый кремовый фон */
--card:#ffffff;      --card-2:#f6f3ed;       /* карточки и вторичный фон */
--ink:#2b2723;       --ink-soft:#6f685f;  --dim:#a39c91;  --line:#ece7df;
--accent:#d98a45;    --accent-soft:#f4e7d6; --accent-ink:#b5702f;  /* тёплый песочный */
--green:#5a9a5e;     --red:#c8624a;         /* рост/падение */
--shadow:0 1px 2px rgba(70,55,40,.04), 0 18px 40px -22px rgba(70,55,40,.22);
--r:22px;            --r2:15px;             /* радиусы карточки/вложенного блока */
```

- **Шрифт:** `Inter`, fallback на системный (`-apple-system,'Segoe UI',Roboto,sans-serif`).
  Допустимо подключить Inter через Google Fonts CDN (как уже делается в `login_page.py`),
  но system-fallback обязателен.
- **Типографика:** крупные **лёгкие** цифры (вес 200–300): время, температура. Заголовки
  карточек — 13px/600, приглушённые подписи — 11–12px `--dim`. Лёгкий отрицательный
  letter-spacing (`-.01em`).
- **Формы:** карточки `--r`, мягкая тень `--shadow`, без рамок. Пилюли/чипы — `border-radius:999px`.
- **Тёмная тема:** включаем (решение §9.1). CSS заложен в мокапе через `html.dark`; добавляем
  кнопку-тумблер в навбаре + сохранение в `localStorage['dd-theme']`, первый заход — по
  `prefers-color-scheme`.

## 2. Макет (bento-grid)

- Контейнер: `display:flex; gap` слева плавающий **pill-навбар**, справа `.main` (контент).
- `.main` сверху — шапка: крупное время, «Доброе утро, {PROFILE_NAME}», строка статуса.
- Сетка карточек: `display:grid; gap:14px; grid-template-columns:repeat(4,1fr)`.
- **Спаны (десктоп):** hero `span 4`; weather `1` + todos `3`; rates `1` + crypto `1` + news `2`;
  mail `2` + recs `2`.
- **Брейкпоинты:**
  - `≤820px` → 2 колонки: hero/todos/news/mail/recs `span 2`; weather/rates/crypto `span 1`; навбар скрыт.
  - `≤480px` → 1 колонка: все карточки `span 1`.
- Точные правила — в `<style>` мокапа, переноси как есть.

## 3. Карточки → маппинг на данные `build_digest()`

`build_digest()` (`app/digest.py`) возвращает dict со следующими ключами. Каждая карточка
обязана корректно отрабатывать состояния **ok / пусто / ошибка**.

| Карточка | data-id | Источник в dict | Поля | Пусто/ошибка |
|---|---|---|---|---|
| Приветствие (hero) | `hero` | `ai.greeting` (str\|None) | текст; бейдж «AI» | если None — показать статичное «Доброе утро» без AI-бейджа |
| Погода | `weather` | `weather` | `city, temp, description`, метео-мета, `hourly[]` (бар-график) | `weather.ok==False` → «Не удалось получить: {error}»; нет `hourly` → без графика |
| Дела | `todos` | `ai.todos_ranked` иначе `todos` | `text, priority(high/med/low), meta` | если список пуст → «Дел нет» |
| Курсы ЦБ | `rates` | `rates` | `items[].{code,value,delta?}` | нет данных → «Нет данных» |
| **Крипто** | `crypto` | **новый источник** (см. §4) | `BTC/ETH/TON`, USD+RUB, %24ч | источник упал → «Нет данных» |
| Новости | `news` | `ai.news_summary` + `news` | summary (AI-box) + `items[].{title,source,time,url}` | нет summary → скрыть AI-box; нет новостей → «Нет данных» |
| Почта | `mail` | `ai.inbox` иначе `mail.emails` | `subject, from, priority` | `mail.ok==False` → «Gmail не настроен»/error |
| Рекомендации | `recs` | `recommendations` (list[str]) | чипы | пусто → скрыть карточку |

**Шапка-статус** («Всё спокойно · N дел · M важных писем»): N = `len(todos)`,
M = число писем с `priority=="high"` (из `ai.inbox`/`mail.emails`). Точка `--green`, если
важных писем нет; иначе `--accent`.

**Приоритеты** (дела и почта) → цветные точки: `high`→`--accent`, `med`→`#c9a45a`, `low`→`--line`.
Списки новостей/почты — строки с шевроном `›` и hairline-разделителями (класс `.row`).

**Экранирование:** весь пользовательский/внешний текст — через существующий `_esc()`
(`html.escape`). Ссылки новостей (`url`) — в `href`, с `rel="noopener"`.

## 4. Новый источник: крипта (CoinGecko)

Создать `app/sources/crypto.py` по образцу `app/sources/currency.py` (через `requests`,
try/except, возврат `{"ok": False, ...}` при ошибке).

- **API (без ключа):**
  `GET https://api.coingecko.com/api/v3/simple/price?ids=bitcoin,ethereum,the-open-network&vs_currencies=usd,rub&include_24hr_change=true`
- **Возврат:**
  ```python
  {"ok": True, "items": [
     {"code": "BTC", "usd": 103240, "rub": 8156000, "change24h": 1.8},
     {"code": "ETH", ...}, {"code": "TON", ...}
  ]}
  ```
- **Конфиг (`.env`/`config.py`):** `CRYPTO_COINS=bitcoin,ethereum,the-open-network`
  (маппинг id→тикер задать в коде: bitcoin→BTC, ethereum→ETH, the-open-network→TON).
  Если `CRYPTO_COINS` пуст — карточку не показывать (как опциональный источник).
- **Wiring:** вызвать в `build_digest()`, добавить ключ `"crypto"` в возвращаемый dict;
  таймаут запроса как у других источников; кэш не обязателен.
- **Graceful degradation:** при ошибке/таймауте — `{"ok": False}`, карточка показывает «Нет данных».

## 5. Интерактив (vanilla JS, из мокапа)

1. **Drag-to-reorder плиток.** HTML5 DnD: каждая `.card` `draggable`, грип `⠿` при hover.
   Порядок сохраняется в `localStorage['dd-order']` (CSV из `data-id`), восстанавливается при загрузке.
   Код — в `<script>` мокапа. *Нюанс:* нативный DnD работает на десктопе; на тач-устройствах
   перетаскивание не сработает (это ок; не блокер). Если нужен мобильный drag — отдельной задачей
   подключить SortableJS с CDN.
2. **Переключатель валюты крипты ₽/$.** Сегментированный тумблер в шапке карточки.
   Оба значения (USD и RUB) приходят из одного запроса CoinGecko и кладутся в
   `data-usd`/`data-rub` на `.num`; переключение — на клиенте, выбор в `localStorage['dd-ccy']`.
   **По умолчанию ₽** (если сохранённого значения нет). Проценты 24ч не меняются при переключении.
3. **Кнопка темы light/dark** (решение §9.1): тогл класса `dark` на `<html>`, сохранение в
   `localStorage['dd-theme']`, первый заход — по `prefers-color-scheme`. Обернуть доступ к
   localStorage в try/catch.
4. **Pill-навбар (слева).** Заменить символы-заглушки (`⌂ ↻ ☷ ⎋`) на нормальные иконки
   (inline SVG, без icon-библиотек). Действия: «домой» (на `/`), «выход» → существующий `/logout`,
   «тема» (см. п.3), «сбросить порядок» (очистка `dd-order`). Кнопки без реальной логики не добавлять.

## 7. Edge cases (обязательно)

- Любой источник упал → его карточка показывает пустое/ошибку, **остальная страница работает**.
- LLM выключен (`ai.enabled==False`) → hero показывает статичное «Доброе утро, {имя}» без AI-бейджа;
  дела берутся из `todos` (не `todos_ranked`); AI-box новостей скрыт.
- Длинные темы писем/заголовки новостей → обрезка по 1–2 строки (`text-overflow:ellipsis`/clamp).
- Пустые списки → дружелюбный плейсхолдер, не пустая карточка с одним заголовком (кроме recs — её скрываем).
- `localStorage` недоступен/выключен → drag, переключатель валют и тема просто не сохраняются
  (`dd-order`/`dd-ccy`/`dd-theme`), но не падают (обернуть в try/catch).
- Без JS → сетка и все данные видны (drag/переключатель/тема — прогрессивное улучшение).
- XSS: всё внешнее через `_esc()`; `url` валидировать (только http/https).

## 8. Тесты и приёмка

Минимальная проверка (как в `CLAUDE.md`):
1. `python -m py_compile main.py app/*.py app/sources/*.py` — без ошибок.
2. `app.test_client()`: `/` без сессии → 302 на `/login`; `/login` → 200 (новый стиль);
   `/` с сессией → 200 и в HTML присутствуют секции (погода/дела/курсы/крипто/новости/почта/рекомендации).
3. Рендер с «упавшими» источниками (передать `{"ok":False}`) → страница рендерится, карточки
   показывают плейсхолдеры (добавить юнит-тест на `render_html` с моками — тестов в проекте пока
   нет, это хороший повод завести `tests/test_render.py`).
4. `crypto.py`: тест на парсинг успешного ответа и на graceful-degradation при ошибке (мок `requests`).
5. Визуально сверить с `docs/dashboard-mockup.html` (light; и dark, если делаем).

**Критерии приёмки:**
- Дашборд визуально соответствует мокапу (палитра, отступы, радиусы, тени, типографика).
- Все 8 карточек берут реальные данные из `build_digest()` (+ новый crypto).
- Drag-to-reorder и переключатель ₽/$ работают и переживают перезагрузку.
- Адаптив 4→2→1 колонка. Graceful degradation подтверждена тестами.
- `py_compile` и `test_client`-проверки зелёные.

## 9. Принятые решения (согласовано с владельцем)

1. **Тёмная тема — кнопка-тумблер light/dark.** В навбаре кнопка переключения,
   выбор сохраняется в `localStorage['dd-theme']` (`light`/`dark`), применяется добавлением
   класса `dark` на `<html>`. При первом заходе (нет сохранённого значения) — следовать
   `prefers-color-scheme`. CSS тёмной темы уже есть в мокапе (`html.dark`).
2. **Страница входа — редизайним сразу в этом же заходе.** `login_page.py` приводим к той же
   дизайн-системе (фон, карточка с QR, типографика, тёмная тема). Логику auth/QR НЕ трогаем —
   только оформление. Тесты `/login` (см. §8) обязательны.
3. **Крипто-валюта по умолчанию — `₽` (рубли).** Если в `localStorage['dd-ccy']` ничего нет,
   стартуем в рублях; переключатель ₽/$ работает как описано в §5.2.
4. **Бар-график погоды — по реальным почасовым.** Расширить `app/sources/weather.py`: добавить
   к запросу Open-Meteo `hourly=temperature_2m` (ключ не нужен), вернуть в dict массив почасовых
   температур (например `weather["hourly"]` — список из ~12 ближайших часов). Карточка рисует
   тонкий бар-график по этим значениям. Если почасовых нет (`ok==False` или пустой массив) —
   график не рисуем, показываем только текущую температуру. **Никаких фейковых/декоративных
   данных.**
5. **Список монет — через `.env` (`CRYPTO_COINS`).** Набор задаётся в конфиге
   (`CRYPTO_COINS=bitcoin,ethereum,the-open-network`); маппинг id→тикер (bitcoin→BTC и т.д.) —
   в коде `crypto.py`. Пустой `CRYPTO_COINS` → карточку крипты не показываем.
